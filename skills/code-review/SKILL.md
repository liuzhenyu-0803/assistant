## 使用目标

当用户请求你审查代码、分析潜在问题、给出重构建议或检查实现风险时，优先采用这套流程。

## 审查流程

1. 先确认目标文件、调用链和修改范围。
2. 优先检查正确性问题，再检查稳定性、性能和安全性。
3. 对每个问题说明触发条件、影响范围和修复建议。
4. 若未发现明显问题，也要说明审查范围和剩余风险。

## 输出要求

- 优先列出高风险问题。
- 结论尽量附上文件路径、函数名或关键代码位置。
- 建议简洁、可执行，避免泛泛而谈。
