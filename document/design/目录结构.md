# 目录结构

本文档定义 Assistant 项目的代码组织、目录规范和命名约定。

## 1. 项目根目录

```
assistant/
├── package.json                # 根 package.json，定义 workspace
├── pnpm-workspace.yaml         # pnpm workspace 配置
├── tsconfig.base.json          # 共享的 TypeScript 基础配置
├── .gitignore
│
├── packages/
│   └── shared/                 # 前后端共享的类型与工具
│
├── client/                     # React 前端应用
│
├── server/                     # Node.js 后端应用
│
├── document/                   # 项目文档（不参与构建）
│   ├── requirements/           # 需求文档
│   ├── design/                 # 设计文档
│   ├── development/            # 开发文档（任务拆解等）
│   └── guides/                 # 指南
│
├── skills/                     # Skill 定义目录（运行时读取）
│   └── {skill-name}/
│       ├── SKILL.md            # Skill 元信息与指令
│       └── ...                 # 附属资源文件
│
└── data/                       # 运行时数据存储（gitignore）
    ├── settings.json
    ├── mcp-servers.json
    └── conversations/
        ├── index.json
        └── {conversationId}/
            ├── conversation.json
            └── attachments/
```

## 5. 命名约定

### 5.1 文件命名

| 类型 | 规则 | 示例 |
|------|------|------|
| React 组件 | PascalCase.tsx | `MessageItem.tsx` |
| Hook | camelCase，use 前缀 | `useSSEStream.ts` |
| Store | kebab-case | `conversation-store.ts` |
| Service / 工具 | kebab-case | `json-store.ts` |
| 类型定义 | kebab-case | `message.ts` |
| 样式模块 | 与组件同名.module.css | `MessageItem.module.css` |

### 5.2 代码命名

| 类型 | 规则 | 示例 |
|------|------|------|
| 接口/类型 | PascalCase | `Conversation`, `MessageContent` |
| 函数 | camelCase | `createConversation()` |
| 常量 | UPPER_SNAKE_CASE | `MAX_CONCURRENT_SUB_AGENTS` |
| 变量 | camelCase | `conversationId` |
| React 组件 | PascalCase | `MessageItem` |
| CSS 类 | camelCase (CSS Modules) | `styles.messageContainer` |

### 5.3 目录命名

- 全部使用 kebab-case
- 单数形式表示单一模块（`agent/`）
- 复数形式表示同类文件集合（`routes/`、`services/`、`components/`）

## 6. 不建议的做法

- 不要在 `routes/` 中写业务逻辑
- 不要在 `storage/` 中引用 `services/`
- 不要在前端直接拼接 API URL，统一通过 `services/` 层
- 不要在组件中直接调用 `fetch`，通过 `services/` 和 `hooks/` 封装
- 不要在 `packages/shared` 中引入仅前端或仅后端的依赖
- 不要把运行时数据文件提交到 Git（`data/` 目录在 `.gitignore` 中）
