# 数据结构

本文档定义 Assistant 的全部数据模型和 JSON 存储文件结构。类型定义使用 TypeScript 格式，前后端通过 `packages/shared` 共享。

## 1. 核心类型

### 1.1 会话 (Conversation)

```typescript
interface Conversation {
  id: string                    // 格式：conv_{nanoid}
  title: string                 // 自动生成的会话标题
  revision: number              // 从 1 起的自增整数，每次消息历史变更时递增
  createdAt: string             // ISO 8601 时间戳
  updatedAt: string             // ISO 8601 时间戳
  lastMessageAt: string | null  // 最后一条消息的时间戳
  messageCount: number          // 消息总数
  fork: Fork | null             // 分支来源信息，非分支会话为 null
  summary: Summary | null       // 上下文摘要，未生成时为 null
}

interface Fork {
  sourceConversationId: string  // 来源会话 ID
  sourceMessageId: string       // 从哪条消息分支
  sourceRevision: number        // 分支时来源会话的 revision
}
```

### 1.2 消息 (Message)

```typescript
interface Message {
  id: string                    // 格式：msg_{nanoid}
  role: 'user' | 'assistant'
  content: MessageContent[]     // 消息内容块数组
  status: MessageStatus
  runId?: string                // 仅助手消息，关联的 Run ID
  createdAt: string             // ISO 8601 时间戳
}

type MessageStatus =
  | 'completed'                 // 正常完成
  | 'interrupted'               // 用户中断，部分内容已保留
  | 'failed'                    // 生成失败

type MessageContent =
  | TextContent
  | AttachmentContent
  | ToolCallContent
  | SubAgentContent

interface TextContent {
  type: 'text'
  text: string
}

interface AttachmentContent {
  type: 'attachment'
  attachment: AttachmentRef
}

interface ToolCallContent {
  type: 'tool-call'
  toolCall: ToolCallRecord
}

interface SubAgentContent {
  type: 'sub-agent'
  subAgent: SubAgentRecord
}
```

### 1.3 附件 (Attachment)

```typescript
interface AttachmentRef {
  id: string                    // 格式：att_{nanoid}
  filename: string              // 存储在磁盘上的文件名（含扩展名）
  originalName: string          // 用户上传时的原始文件名
  mimeType: string              // MIME 类型
  size: number                  // 文件大小（字节）
  status: AttachmentStatus
  createdAt: string             // ISO 8601 时间戳
}

type AttachmentStatus =
  | 'staged'                    // 已上传暂存，尚未绑定消息
  | 'attached'                  // 已绑定到消息
  | 'copying'                   // 分叉时正在复制
  | 'unavailable'               // 文件丢失或不可用
```

**孤儿清理规则：**

- `staged` 状态的附件可能因用户关闭页面而遗留为孤儿文件
- **启动时清理：** 服务端启动时扫描全部会话附件目录，删除未被任何消息引用的暂存附件
- **运行时清理：** 用户连接断开后，等待 5 分钟后清理该会话中未绑定消息的暂存附件（容忍页面刷新等短暂断连）

### 1.4 工具调用记录 (ToolCallRecord)

```typescript
interface ToolCallRecord {
  id: string                    // 格式：tc_{nanoid}
  toolName: string              // 工具名称
  serverId: string              // MCP Server ID
  arguments: Record<string, unknown>  // 工具调用参数
  result?: string               // 工具返回结果
  status: 'running' | 'success' | 'failed' | 'cancelled'
  error?: string                // 失败时的错误信息
  startedAt: string
  completedAt?: string
}
```

### 1.5 子代理记录 (SubAgentRecord)

```typescript
interface SubAgentRecord {
  id: string                    // 格式：sa_{nanoid}
  task: string                  // 任务描述
  summary?: string              // 摘要结果
  detail?: string               // 完整执行详情（可展开查看）
  status: 'running' | 'success' | 'failed' | 'timeout' | 'cancelled'
  error?: string                // 失败时的错误信息
  startedAt: string
  completedAt?: string
}
```

### 1.6 上下文摘要 (Summary)

```typescript
interface Summary {
  text: string                  // 摘要文本
  upToMessageId: string         // 摘要覆盖到的最后一条消息 ID
  createdAt: string             // ISO 8601 时间戳
}
```

### 1.7 Run

Run 是运行时概念，不独立持久化，但其 ID 记录在助手消息上。运行时由 `RunService` 在内存中以 `Map<string, Run>` 维护活跃 Run 状态，Run 完成或中断后从 Map 中移除。服务端重启时内存中无活跃 Run，已持久化的助手消息通过 `runId` 字段保留关联关系。

```typescript
interface Run {
  id: string                    // 格式：run_{nanoid}
  conversationId: string
  status: 'running' | 'completed' | 'interrupted' | 'failed'
  messageId: string             // 关联的助手消息 ID
  startedAt: string
  completedAt?: string
}
```

## 2. 配置类型

### 2.1 全局设置 (Settings)

```typescript
interface Settings {
  baseURL: string               // OpenAI 兼容 API 的基础 URL
  apiKey: string                // API 密钥（存储时明文，返回前端时脱敏）
  model: string                 // 默认模型名称
  contextWindowSize: number     // 上下文窗口大小（token 数），默认 128000
}
```

### 2.2 MCP Server 配置

```typescript
interface MCPServerConfig {
  id: string                    // 格式：mcp_{nanoid}
  name: string                  // 显示名称
  transport: 'stdio' | 'streamable-http' | 'sse'
  enabled: boolean

  // stdio transport
  command?: string              // 可执行命令
  args?: string[]               // 命令参数
  env?: Record<string, string>  // 环境变量

  // streamable-http / sse transport
  url?: string                  // 远程服务 URL
  headers?: Record<string, string>  // 自定义请求头
}
```

### 2.3 Skill 元信息

```typescript
interface SkillMeta {
  name: string                  // Skill 名称
  description: string           // 功能简述
  match: string[]               // 适用场景关键词或模式列表
  path: string                  // Skill 目录相对路径
  contentPath: string           // SKILL.md 文件路径
}
```

SKILL.md 文件格式示例：

```markdown
---
name: code-review
description: 代码审查与质量分析
match:
  - 代码审查
  - review
  - code review
  - 代码质量
---

## 审查流程

1. 先阅读代码整体结构
2. 检查潜在的安全问题
3. 检查代码风格一致性
...
```

## 3. 存储文件结构

### 3.1 存储目录布局

```
data/
├── settings.json               # 全局设置
├── mcp-servers.json            # MCP Server 配置
└── conversations/
    ├── index.json              # 会话索引
    └── {conversationId}/
        ├── conversation.json   # 会话元信息 + 消息
        ├── conversation.json.bak  # 备份
        └── attachments/
            ├── a1b2c3d4.png    # 附件文件（使用生成的文件名）
            └── e5f6g7h8.pdf
```

### 3.2 settings.json

```json
{
  "baseURL": "https://api.openai.com/v1",
  "apiKey": "sk-xxxxxxxxxxxxxxxx",
  "model": "gpt-4o",
  "contextWindowSize": 128000
}
```

### 3.3 mcp-servers.json

    {

### 3.4 conversations/index.json

索引文件仅存储会话列表的摘要信息，用于快速加载列表而无需读取每个会话文件：

```json
{
  "conversations": [
    {
      "id": "conv_abc123",
      "title": "如何实现一个Web助手",
      "createdAt": "2025-01-01T00:00:00.000Z",
      "updatedAt": "2025-01-01T01:00:00.000Z",
      "lastMessageAt": "2025-01-01T01:00:00.000Z",
      "messageCount": 12
    }
  ]
}
```

## 4. ID 生成规则

| 类型 | 前缀 | 格式 | 示例 |
|------|------|------|------|
| 会话 | conv_ | conv_{nanoid(12)} | conv_V1StGXR8_Z5j |
| 消息 | msg_ | msg_{nanoid(12)} | msg_dBjftJeZ4yGg |
| 附件 | att_ | att_{nanoid(12)} | att_7V1s3kYxPqN2 |
| Run | run_ | run_{nanoid(12)} | run_Xf3aB7cD9eGh |
| 工具调用 | tc_ | tc_{nanoid(12)} | tc_mK4nL8pQ2rSt |
| 子代理 | sa_ | sa_{nanoid(12)} | sa_uV6wX0yA3bCd |
| MCP Server | mcp_ | mcp_{nanoid(12)} | mcp_eF5gH7iJ1kLm |

所有 ID 全局唯一，使用 nanoid 生成。
